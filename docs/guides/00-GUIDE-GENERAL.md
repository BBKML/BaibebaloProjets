# Guide général – Plateforme Baibebalo

## À qui s’adresse ce guide

Ce document présente la plateforme Baibebalo dans son ensemble. Il est utile pour **tous les utilisateurs** (clients, restaurants, livreurs) et pour la **formation** des nouveaux partenaires.

---

## 1. Qu’est-ce que Baibebalo ?

Baibebalo est une **plateforme de livraison de repas** (et de courses) qui relie :

- **Les clients** : ils commandent depuis l’application mobile client.
- **Les restaurants** : ils reçoivent les commandes, préparent les plats et les remettent aux livreurs.
- **Les livreurs** : ils récupèrent les commandes au restaurant et les livrent aux clients.

La plateforme couvre notamment la zone de **Korhogo** (Côte d’Ivoire) et peut s’étendre à d’autres villes.

---

## 2. Rôles et applications

| Rôle        | Application        | Rôle principal |
|------------|--------------------|-----------------|
| **Client** | App « Baibebalo » (client) | Commander des repas, suivre la livraison, payer. |
| **Restaurant** | App « Baibebalo Restaurant » | Gérer le menu, accepter/refuser les commandes, préparer, communiquer avec le client. |
| **Livreur** | App « Baibebalo Livreur » | Être disponible, accepter des courses, aller au restaurant puis chez le client, confirmer la livraison. |
| **Admin** | Interface web Admin | Gérer la plateforme (restaurants, livreurs, commandes, finances, paramètres). |

---

## 3. Parcours type d’une commande

1. **Client** : choisit un restaurant, compose son panier, valide l’adresse et le paiement, envoie la commande.
2. **Restaurant** : reçoit la commande (notification + app), l’accepte ou la refuse. S’il accepte, il prépare les plats.
3. **Plateforme** : affecte un livreur disponible (ou le livreur accepte la course).
4. **Livreur** : se rend au restaurant, récupère la commande, part chez le client, remet le colis et confirme la livraison (éventuellement avec code de confirmation ou photo).
5. **Client** : reçoit sa commande et peut noter le restaurant / le livreur.

---

## 4. Paiement

- **Carte / Mobile Money** : le client peut payer en ligne (selon configuration).
- **Paiement à la livraison** : le client paie le livreur (espèces ou Mobile Money). Le livreur enregistre le paiement dans l’app et effectue les remises selon les consignes de la plateforme.

Les **restaurants** sont réglés selon les termes du contrat (commission plateforme, délais de paiement, etc.).

## 6. Bonnes pratiques communes

- **Connexion internet** : garder une connexion stable pour recevoir les commandes et les mises à jour en temps réel.
- **Notifications** : activer les notifications de l’app pour ne pas manquer les nouvelles commandes (restaurant) ou les nouvelles courses (livreur).
- **Réactivité** : accepter ou refuser les commandes rapidement (restaurant) et traiter les courses dans les délais (livreur).
- **Communication** : utiliser le chat intégré en cas de question (client–restaurant, client–livreur) et signaler tout incident (adresse incorrecte, client absent, etc.).
